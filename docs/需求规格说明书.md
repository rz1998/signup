## 1. 项目概述

### 1.1 项目名称
**报名系统**（Signup System）

### 1.2 项目简介
报名系统是一个面向地推团队和在线运营团队引流报名的 SaaS 管理平台，支持多租户架构、活动管理、自定义报名字段、来源追踪和数据统计。

### 1.3 项目背景
地推团队和运营团队需要一个统一的平台来管理活动报名数据，包括创建活动、配置报名字段、追踪营销人员推广效果、管理报名信息。

### 1.4 目标用户
| 用户类型 | 说明 |
|---------|------|
| 系统管理员 | 平台运营方，管理所有租户 |
| 公司管理员 | 企业用户，管理本公司及下属分支机构 |
| 分支机构管理员 | 管理本机构的营销人员和活动 |
| 营销管理人员 | 团队负责人，管理普通营销人员和活动 |
| 普通营销人员 | 地推/运营人员，分享报名页面给游客 |
| 游客 | 潜在客户，通过二维码/链接填写报名信息 |

## 2. SaaS 多租户架构

### 2.1 层级结构

```
系统（System）
└── 公司（Company）
    └── 分支机构（Branch）
        └── 营销人员（Marketing Staff）
            ├── 营销管理人员（Manager）
            └── 普通营销人员（Sales）
```

### 2.2 角色权限定义

| 角色 | 层级 | 权限范围 |
|------|------|---------|
| 系统管理员 | 系统级 | 管理所有公司和分支机构，可创建、编辑、删除公司和分支机构；管理所有用户 |
| 公司管理员 | 公司级 | 管理本公司信息（仅编辑，不可创建/删除本公司）；管理本公司下所有分支机构；管理本公司内所有用户 |
| 分支机构管理员 | 分支机构级 | 管理本分支机构信息（编辑/删除/更新本分支机构）；管理本分支机构的所属用户 |
| 营销管理人员 | 营销团队级 | 管理本团队的普通营销人员；可创建和管理本团队的活动（分支机构层面活动） |
| 普通营销人员 | 个人级 | 分享报名页面/二维码，查看自己名下的报名数据 |
| 游客 | 无 | 填写报名信息，查看/修改已保存信息 |

### 2.3 租户隔离策略

- **数据隔离**：所有业务数据按 `company_id` 和 `branch_id` 进行逻辑隔离
- **权限控制**：用户只能查看和操作其所属公司及分支机构下的数据
- **系统管理员**：可跨租户操作所有数据
- **查询过滤**：所有列表查询默认附加租户过滤条件

### 3.3 活动管理模块

#### 3.3.1 功能描述
管理营销活动，包括创建活动、配置活动页面样式和管理活动状态。

#### 3.3.2 活动层级设计

活动支持两个层级的归属：

| 活动层级 | 创建者 | 可见范围 | 说明 |
|---------|-------|---------|------|
| 公司层面活动 | 系统管理员、公司管理员 | 整个公司及所有下属分支机构 | 属于公司层面的统一活动，所有分支机构可见 |
| 分支机构活动 | 分支机构管理员、营销管理人员 | 本分支机构 | 属于特定分支机构的活动，仅本机构可见 |

**约束规则**：
- 活动必须归属到公司或分支机构，即 `company_id` 和 `branch_id` 不能同时为空
- 如果设置了 `branch_id`，则 `company_id` 由系统自动填充（所属公司）
- 如果设置了 `company_id`（公司层面活动），则 `branch_id` 必须为空

#### 3.3.3 用户故事
- 作为公司管理员，我希望创建一个公司层面的统一活动，以便所有分支机构共同参与
- 作为营销管理人员，我希望创建一个分支机构活动，以便开展本机构的报名工作
- 作为营销管理人员，我希望配置活动页面样式（图片、文字、背景），以便吸引更多报名
- 作为营销管理人员，我希望管理活动状态（报名中/报名结束），以便控制报名窗口

#### 3.3.3 业务流程
```
创建活动：填写活动基本信息 → 配置活动页面样式 → 保存草稿 → 发布
编辑活动：选择活动 → 修改信息/样式 → 保存
变更状态：选择活动 → 修改状态（草稿→报名中→报名结束）→ 保存
删除活动：选择活动 → 确认删除 → 软删除
```

#### 3.3.4 功能清单
- 活动列表（支持按状态筛选、按名称搜索）
- 创建活动
- 编辑活动信息（名称、描述、时间、地点、人数上限）
- 配置活动页面：
  - 页面标题、副标题
  - 背景颜色/背景图片
  - 宣传图片（最多5张）
  - 宣传文字
  - 报名说明
  - 联系方式（电话、邮箱）
- 管理活动状态：草稿（draft）、报名中（active）、报名结束（closed）、已取消（cancelled）、已完成（completed）
- 删除活动（软删除）

```
游客修改：进入报名记录 → 选择报名 → 修改信息 → 保存
管理员审核：查看报名列表 → 选择报名 → 审核（确认/拒绝）→ 保存状态
```

- 查看报名详情
- 新增报名（游客）
- 编辑报名（游客，限报名中状态的活动）
- 报名状态管理：待审核（pending）、已确认（confirmed）、已拒绝（rejected）、已取消（cancelled）

#### 3.5.5 报名数据存储
报名信息需保存以下关键数据：
- 活动ID（activity_id）
- 分支机构ID（branch_id）
- 营销人员ID（sales_id，用于追踪来源）
- 游客OpenID（visitor_openid）
- 表单数据（form_data，JSON格式）
- 报名状态（status）

### 3.6 分享模块

#### 3.6.1 功能描述
生成活动报名页面链接和二维码，追踪营销效果。

#### 3.6.2 用户故事
- 作为营销管理人员，我希望生成带有自己标识的报名链接/二维码，以便分享给潜在客户
- 作为系统，我希望追踪每个链接/二维码的访问次数和报名转化率

#### 3.6.3 业务流程
```
生成分享：选择活动 → 点击生成链接/二维码 → 系统生成唯一标识 → 返回分享链接/二维码
分享传播：营销人员分享链接/二维码 → 游客扫码 → 系统记录访问来源
数据追踪：系统记录访问次数、分享次数 → 关联报名数据 → 统计转化率
```

#### 3.6.4 功能清单
- 生成活动报名页面链接（带营销人员标识）
- 生成活动报名二维码（带营销人员标识）
- 查看分享记录（分享链接/二维码列表）
- 追踪访问次数（visit_count）
- 追踪报名转化

#### 3.6.5 分享链接格式
```
{BaseURL}/visitor/activity/{shareId}
示例：http://192.168.8.230:8082/visitor/activity/abc123
```

### 4.1 实体关系图

```
users ──┬── belongs to ──→ companies
        └── belongs to ──→ branches ── belongs to ──→ companies
```
